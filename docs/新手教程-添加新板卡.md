# 芯片适配新手教程 (Adding New Board Tutorial)

本文档旨在为初学者提供一份傻瓜式的向导，演示如何在 EmbedBlocks Studio 源码中添加一款新的开发板支持。

> [!NOTE]
> **原理**: 我们将利用 `src/config/custom` 目录的 **自动发现机制**。你不需要修改任何核心代码，只需要创建一个 TypeScript 配置文件。

---

## 1. 准备工作 (Preparation)

1.  **实物准备**: 手边有一块你想要添加的开发板（如 ESP32-C3 SuperMini）。
2.  **资料准备**:
    *   **引脚图 (Pinout)**: 知道哪个物理引脚对应 GPIO 几。
    *   **构建参数**: 知道它在 PlatformIO 或 Arduino IDE 中选什么型号（如 `esp32-c3-devkitm-1`）。

---

## 2. 第一步：创建配置文件 (Step 1: Create Config File)

所有的“实验性”或“私有”板子都放在 `src/config/custom/` 目录下。

1.  打开 `src/config/custom/` 文件夹。
2.  新建一个文件：`my_new_board.ts` (文件名随意，但必须以 `.ts` 结尾)。

---

## 3. 第二步：编写核心配置 (Step 2: Define Board)

将以下模版复制到你的文件中：

```typescript
import { defineBoard } from '../../utils/extension_helpers';

// 使用 defineBoard 助手函数，会有智能代码提示！
export const MY_BOARD = defineBoard({
    // [必填] 唯一 ID，不要和现有板子冲突
    id: 'my_c3_mini',
    
    // [必填] 显示名称 (推荐支持中英文)
    name: { 
        zh: '我的 C3 Mini 开发板', 
        en: 'My C3 Mini Board' 
    },

    // [必填] 硬件参数 (用于 UI 显示)
    mcu: 'ESP32-C3',
    freq: '160MHz',
    flash: '4MB',
    ram: '400KB',
    fqbn: 'platformio:esp32-c3-devkitm-1', // 格式: platform:board_id

    // [必填] 构建参数 (传递给 PlatformIO 的关键配置)
    // 如果你不确定填什么，请先在 PlatformIO 中找到能跑通的配置
    build: {
        envName: 'c3_mini_env',       // 编译环境名，保持唯一即可
        platform: 'espressif32',      // 平台名
        board: 'esp32-c3-devkitm-1',  // PIO 板型 ID
        framework: 'arduino'          // 框架
    },

    // [必填] 引脚定义 (见下一步)
    pins: {
        digital: [],
        analog: [],
        pwm: [],
        i2c: [],
        spi: [],
        serial: []
    }
});
```

---

## 4. 第三步：定义引脚映射 (Step 3: Pin Mapping)

引脚定义决定了用户在积木的下拉菜单中能看到哪些选项。

**格式**: `{ label: '显示的文字', value: '生成的代码' }`

修改 `pins` 部分如下：

```typescript
    pins: {
        // 数字引脚：通常列出所有 GPIO
        digital: [
            { label: 'P0 (LED)', value: '0' },
            { label: 'P1 (Button)', value: '1' },
             // ... 更多引脚
        ],

        // 模拟引脚：列出支持 ADC 的引脚
        analog: [
            { label: 'A0 (P0)', value: '0' }
        ],

        // PWM：ESP32 系列通常全引脚支持，写 'any' 即可
        pwm: [
            { label: '任意引脚', value: 'any' }
        ],

        // 通讯接口：用于 I2C / SPI 积木默认选中的引脚
        i2c: [
            { label: 'SDA (P8)', value: '8' },
            { label: 'SCL (P9)', value: '9' }
        ],
        spi: [], // 如果不用可以留空

        // 串口：定义硬件串口
        serial: [
            { label: 'USB Serial', value: 'Serial' },
            { label: 'UART1', value: 'Serial1' }
        ]
    }
```

> [!TIP]
> **多语言标签**: 如果你想让引脚名字也支持中英文切换，可以这样写：
> `label: { zh: '板载 LED', en: 'Onboard LED' }`

---

## 5. 第四步：声明硬件能力 (Step 4: Capabilities)

这一步可选，但推荐添加。这决定了工具箱里会出现哪些分类。
例如，如果你的板子没有蓝牙，就不要开启蓝牙分类。

```typescript
    // 在 defineBoard 的参数对象中添加：
    capabilities: {
        wifi: true,      // 开启 Wi-Fi 相关积木
        bluetooth: true, // 开启 蓝牙 相关积木
        sd: false,       // 这是一个没有 SD 卡槽的板子
        ir_remote: true  // 支持红外
    }
```

---

## 6. 第五步：验证与测试 (Step 5: Verification)

1.  **保存文件**: 此时开发服务器 (`npm run dev`) 应该会自动重载。
2.  **查看界面**:
    *   点击顶部栏的开发板选择菜单。
    *   滚动到底部，找到 **"芯片实验室 (Chip Lab)"** -> **"单板仓库"**。
    *   你应该能看到 **"我的 C3 Mini 开发板"**。
3.  **测试编译**:
    *   选中该板子。
    *   拖入一个简单的 "点灯" 程序。
    *   点击 "编译 (Verify)" 按钮。
    *   观察控制台，确保 PlatformIO 能够成功初始环境并开始编译。

---

## 7. 进阶：如何添加一个系列？ (Advance)

如果你有一套板子（比如 V1, V2, Pro 版），不想一个个散落在列表里，可以创建一个文件夹：

1.  新建 `src/config/custom/my_series/` 文件夹。
2.  新建 `index.ts`。
3.  导出 `BoardSeries` 结构：

```typescript
import { BoardSeries } from '../../../types/board';
import { MY_BOARD_V1 } from './board_v1'; // 假设你把配置拆分到了单独文件
import { MY_BOARD_V2 } from './board_v2';

export const MY_SERIES: BoardSeries = {
    id: 'my_cool_series',
    name: { zh: '我的酷炫系列', en: 'My Cool Series' },
    boards: [
        MY_BOARD_V1,
        MY_BOARD_V2
    ]
};
```

系统会自动识别 `boards` 数组，并在菜单中为你创建一个独立的一级分类！

## 总结

*   **路径**: `src/config/custom/*.ts`
*   **工具**: `defineBoard`
*   **核心**: `id`, `name`, `build`, `pins`
*   **生效**: 自动发现，无需注册
