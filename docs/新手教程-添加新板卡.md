# 芯片适配新手教程 (Adding New Board Tutorial)

本文档旨在为初学者提供一份傻瓜式的向导，演示如何在 EmbedBlocks Studio 中快速添加一款个人开发板或特定的芯片支持。

> [!IMPORTANT]
> **原理**: 我们现在采用 **JSON 驱动架构**。你不需要编写任何 TypeScript 代码或进行编译，只需在指定目录创建一个 `.json` 配置文件，软件启动时会自动发现并加载它。

---

## 1. 准备工作 (Preparation)

1.  **确定参数**: 知道它在 PlatformIO 中对应的 `platform` 和 `board` ID（例如：`platform: espressif32`, `board: esp32-c3-devkitm-1`）。
2.  **获取引脚图**: 准备好芯片的 Pinout 图，确定哪些引脚支持数字读写、模拟输入等。

---

## 2. 第一步：创建 JSON 文件 (Step 1: Create JSON)

所有的自定义板卡都存放在 `src/data/boards/custom/` 目录下。

1.  打开 `src/data/boards/custom/` 文件夹。
2.  新建一个文件：`my_awesome_board.json` (文件名建议全英文小写)。

---

## 3. 第二步：编写配置内容 (Step 2: Define Content)

将以下模板复制到你的 JSON 文件中，并根据实际情况修改：

```json
{
  "id": "my_unique_c3_id",
  "name": {
    "zh": "我的 C3 实验板",
    "en": "My Awesome C3 Board"
  },
  "mcu": "ESP32-C3",
  "platform": "espressif32",
  "fcpu": 160000000,
  "specs": "4M Flash / 400K RAM",
  "description": "基于 ESP32-C3 的自定义开发板",
  "pins": {
    "digital": [
      { "label": "D0 (LED)", "value": "0" },
      { "label": "D1 (Button)", "value": "1" }
    ],
    "analog": [
      { "label": "A0 (GPIO 0)", "value": "0" }
    ],
    "pwm": [
      { "label": "PWM (GPIO 0)", "value": "0" }
    ],
    "i2c": [
      { "label": "SDA (GPIO 8)", "value": "8" },
      { "label": "SCL (GPIO 9)", "value": "9" }
    ],
    "spi": [],
    "serial": [
      { "label": "USB Serial", "value": "Serial" }
    ]
  },
  "capabilities": {
    "wifi": true,
    "bluetooth": true
  }
}
```

### 关键字段说明：
- **`id`**: 必须全局唯一，不能与内置板卡冲突。
- **`platform`**: 必须与 PlatformIO 的平台名一致。
- **`fcpu`**: 主频，单位为 **Hz**（例如 160MHz 写成 `160000000`）。
- **`pins`**: 决定了积木下拉菜单中显示的内容。`label` 是你在界面看到的文字，`value` 是最终生成的代码。

---

## 4. 第三步：验证效果 (Step 3: Verification)

1.  **保存并重启**: 保存 JSON 文件后，重启 EmbedBlocks Studio。
2.  **查找板卡**:
    *   在“新建项目”弹窗中，选择底部的 **“自定义 (Custom)”** 分类。
    *   你应该能看到刚刚添加的“我的 C3 实验板”。
3.  **测试积木**: 选择该板卡后，拖入“数字写入”积木，检查下拉菜单是否出现了你定义的 `D0 (LED)` 和 `D1 (Button)`。

---

## 5. 进阶：如何获取内置板卡 ID？

如果你想基于某个官方支持的板卡做修改，可以参考以下目录中的现有文件：
- **STM32**: 存放于 `src/data/boards/stm32/[系列]/` (如 `STM32F1/STM32F103C8.json`)。
- **主流开发板 (Uno/ESP32)**: `src/data/boards/standard/`
- **自定义板卡**: `src/data/boards/custom/`

复制其中的内容到 `custom` 目录并修改 `id` 即可。

---

## 6. 特色：STM32 芯片视觉展示 (STM32 Chip Visuals)

对于 STM32 芯片，EmbedBlocks Studio 提供了一套强大的**即时渲染引擎**，能够将抽象的 JSON 定义转化为高精度的实物级示意图。

### 功能特性
*   **多封装支持**: 完美支持从 `LQFP/QFN` 到复杂的 `TFBGA/UFBGA/WLCSP` 球栅阵列封装。
*   **官方数据驱动**: 系统支持直接加载官方精确的物理引脚坐标 (`pinMap`)，确保 Every Pin in Right Place。
*   **交互式浏览**:
    *   **深度缩放**: 在 BGA 等高密度封装下，放大视图会自动显示微缩引脚编号 (如 "A1", "C5")。
    *   **引脚映射**: 鼠标悬停在引脚上，会实时显示其对应的功能定义 (如 `PA9/USART1_TX`)，辅助硬件接线。

### 配置方式
在 JSON 中提供 `pinMap` 字段即可激活精确模式：
```json
"pinMap": [
  { "name": "PA0", "position": "10" },
  { "name": "PA1", "position": "11" }
]
```
> **提示**: 官方支持的 STM32 芯片已通过脚本自动集成了这些数据，无需手动填写。手动添加新板卡时，如果没有 `pinMap`，系统将自动使用启发式算法生成标准的矩形示意图 (TSSOP/LQFP 等)。

## 总结

*   **路径**: `src/data/boards/custom/*.json`
*   **生效**: 只需重启软件。
*   **优势**: 无需开发环境，直接修改 JSON 即可调整硬件定义。
