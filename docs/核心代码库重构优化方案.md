# 核心代码库优化方案：模块化重构 (Module Refactoring Plan)

**原则**: High Cohesion (高内聚), Low Coupling (低耦合)
**目标**: 解决 `src/modules` 下单文件积木定义过长、逻辑混合的问题，为 Phase 1 (TinyML) 功能扩展扫清障碍。

---

## 1. 现状问题诊断

经过代码审计，发现 `src/modules/core/tinyml.ts` (以及 `variables.ts`) 存在典型的 "God File" 倾向：

*   **混合关注点 (Mixed Concerns)**: UI 定义 (`this.appendDummyInput`) 与 业务逻辑 (`arduinoGenerator.functions_`) 混杂在同一个闭包中。
*   **可维护性差**: 修改 C++ 模板需要在一堆 UI 代码中寻找字符串拼接。
*   **复用性低**: 积木的定义无法被 JSON 格式导出用于其他工具。

## 2. 优化方案：标准模块架构 (SMA)

建议将所有核心模块重构为以下**三层目录结构**：

```text
src/modules/core/tinyml/
├── index.ts        <-- [Entry] 模块元数据导出
├── blocks.ts       <-- [View] 纯粹的 Blockly 积木外观定义
└── generator.ts    <-- [Logic] 纯粹的 C++ 代码生成逻辑
```

### 2.1 职责分离
*   **blocks.ts**: 只负责 `init()`, `onchange()`, `mutator`。不引用 `arduinoGenerator`。
*   **generator.ts**: 只负责 `arduinoGenerator.forBlock[...]`。不引用 `Blockly` UI 类。
*   **index.ts**: 负责将两者粘合，并导出 `BlockModule` 对象。

## 3. Phase 1 重构演示 (以 TinyML 为例)

我们将把当前的 `tinyml.ts` 拆解。

### Step 1: `blocks.ts`
```typescript
import * as Blockly from 'blockly';
import { COLOUR_AI } from '../../constants'; // 从硬编码 280 变为常量

export const defineBlocks = () => {
    Blockly.Blocks['tinyml_knn_init'] = {
        init: function() {
            this.appendDummyInput().appendField(Blockly.Msg.ARD_TINYML_KNN_INIT);
            // ...
            this.setColour(COLOUR_AI);
        }
    };
    // ... 其他积木
};
```

### Step 2: `generator.ts`
```typescript
import { arduinoGenerator, Order } from '../../generators/arduino-base';

export const defineGenerators = () => {
    arduinoGenerator.forBlock['tinyml_knn_init'] = function(block) {
        // 纯粹的 C++ 逻辑，清晰易读
        arduinoGenerator.functions_['knn_struct'] = `struct KNNPoint { ... };`;
        return '';
    };
};
```

### Step 3: `index.ts`
```typescript
import { defineBlocks } from './blocks';
import { defineGenerators } from './generator';
import { BlockModule } from '../../registries/ModuleRegistry';

const init = () => {
    defineBlocks();
    defineGenerators();
};

export const TinyMLModule: BlockModule = {
    id: 'core.tinyml',
    init
};
```

## 4. 预期收益

1.  **耦合度降低**: Generator 不再依赖 UI 加载顺序，Blocks 可以在 Headless 环境（如沙箱）中独立加载。
2.  **开发效率**: 前端专注于 `blocks.ts` 调整 UI，嵌入式工程师专注于 `generator.ts` 优化 C++ 算法，互不冲突。
3.  **扩展性**: 为 Phase 1 的 "Serial Data Sampler" 复杂积木预留了干净的代码空间。
