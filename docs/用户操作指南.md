# EmbedBlocks Studio 综合使用手册 (The Complete Handbook)

欢迎来到嵌入式开发的精彩世界！**EmbedBlocks Studio** 不仅仅是一个编程工具，它是您连接现实世界与数字逻辑的桥梁。本手册将带您从零开始，逐步成长为一名自主的创客。

---

## 第一章：硬件实验室 (Hardware Setup)

在编写第一行代码前，我们需要确保“地基”已经打好。

### 1.1 开发板初探
嵌入式开发板就像一个精简版的电脑大脑。
-   **处理器 (MCU)**: 处理逻辑的中心。
-   **引脚 (Pins)**: 大脑的手脚，用于感受世界（传感器）或改变世界（电机、LED）。
-   **USB 接口**: 既是供电通道，也是程序传输的“高速公路”。

### 1.2 连接与驱动
1.  **物理连接**: 使用优质数据线将开发板接入电脑。
2.  **驱动安装**:
    -   **CH340/CH341**: 常见于国产 Uno/ESP32 模块。
    -   **CP2102**: 常见于中高端开发模块。
> [!TIP]
> 如果顶部工具栏的“串口”列表为空，通常是因为驱动未安装或线缆仅支持充电不支持数据传输。

## 第二章：软件界面全景 (UI Tour)

熟悉您的工具箱是提高效率的第一步。

### 2.1 顶级工具栏 (Top Toolbar)
-   **项目管理**: 新建、打开、保存项目的快捷入口。
-   **硬件选择**: 核心下拉框。在这里选择您手中的板卡型号（决定了积木的兼容性）。如需添加新板子，请参考 [5.2 节](#52-添加自定义开发板)。
-   **串口选择**: 程序上传的终点站。
-   **编译与上传**: 
    -   **编译 (Check)**: 检查语法是否正确，不传输。
    -   **上传 (Arrow)**：先编译，成功后自动通过串口推送到开发板。

### 2.2 积木中心 (Workspace)
-   **左侧工具箱**: 分门别类存放着所有功能模块。
-   **磁吸效果**: 积木边缘的凹凸形状代表了逻辑的连接可能。

### 2.3 右侧辅助面板 (Info Panel)
-   **代码预览**: 实时生成的 C++ 源代码。这是从积木跨越到高级编程的最佳阶梯。
-   **调试输出**: 编译时的报错信息会在这里显示（红色文字通常代表问题所在）。

## 第三章：编程逻辑百科 (Logic & Coding)

### 3.1 程序的生命周期
-   **Setup (初始化)**: 
    -   **意义**: 声明引脚是用来“读”还是“写”，设置初始速度。
    -   **类比**: 入校登记，一生只做一次。
-   **Loop (主循环)**:
    -   **意义**: 真正的逻辑发生地。
    -   **类比**: 每天的课程，上完一遍接着上下一遍。

### 3.2 信号的本质
-   **数字信号 (Digital)**: 只有“开(1)”和“关(0)”。如同电灯开关。
-   **模拟信号 (Analog)**: 具有 0 到 1023 的连续变化。如同音量旋钮。
-   **PWM (呼吸灯原理)**: 通过快速开关模拟中间亮度。

## 第四章：实战案例库 (Tutorials)

### 案例 1：基础 Blink —— 逻辑的初鸣
**目标**: 控制板载 LED 1 秒闪烁。
1.  **积木**: `数字输出(13, 高)` -> `延迟(1000)` -> `数字输出(13, 低)` -> `延迟(1000)`。
2.  **核心**: 为什么要有第二个延迟？**如果没有，LED 灭掉后会瞬间进入下一轮循环变亮，肉眼无法察觉熄灭。**

### 案例 2：串口对话 —— 赋予硬件“语言”
**目标**: 在电脑屏幕上打印实时的传感器读数。
1.  **积木**: `打印到串口(文本: Hello World)`。
2.  **观察**: 打开底部串口监视器，点击“开启串口”。
3.  **注意**: 波特率（Baud Rate）默认通常为 `115200` 或 `9600`。如果不一致，您将看到“乱码”。

### 案例 3：智能控制 (结合条件分支)
**目标**: 当光线暗时自动开灯。
1.  **逻辑**: `如果 (模拟读取(A0) < 500) 执行 { 数字输出(13, 高) } 否则 { 数字输出(13, 低) }`。

## 第五章：高手秘籍 (Pro Tips)

-   **搜索技巧**: 忘记积木在哪？按下 `Ctrl + F` 全局搜索。支持中英文双语模糊匹配。
-   **一键恢复**: 软件意外退出？再次打开时请留意顶部的“恢复备份”提示。
-   **性能调优**: 在设置中调低自动备份频率（如 5s），可以减少低配电脑的卡顿。

### 5.1 多语言环境 (Multi-language Support)

EmbedBlocks 支持 **简体中文** 与 **English** 自动切换。
- **切换方法**: 点击左下角“设置 (Gear)” -> “Gerneral (常规)” -> “Language (语言)”。
- **全翻译覆盖**: 核心 UI、串口监视器、全局搜索及帮助文档均支持实时语言切换。
- **插件翻译**: 若您导入的第三方插件包含语言包，积木标签也将随系统语言自动改变。

### 5.2 添加自定义开发板 (Custom board support)
如果您购买了一块冷门的开发板，或者开发者提供了专用的芯片支持包：
1.  **获取扩展包**: 确保您拥有一个包含 `manifest.json` 和 `board.json` 的文件夹或 `.zip`文件。
2.  **导入扩展**: 
    -   点击侧边栏的 **“扩展 (Puzzle 按钮)”**。
    -   点击 **“Import Extension (从本地导入)”** 并选择该文件夹。
    > [!TIP]
    > **导入提示**: 即使您的插件文件夹解压后多了一层（如 `my-plugin/my-plugin/manifest.json`），系统也会智能尝试寻找并安装，无需您手动整理文件夹。
    > **遇到困难?** 点击界面上的 **"查看发布指南"** 按钮，可以直接打开详尽的本地文档。
3.  **开始使用**: 
    -   点击 “新建项目”。
    -   在开发板下拉列表中，您将立即看到标有 **(Ext)** 后缀的新板子。
    -   选择它，即可针对该硬件开始编程。

## 第六章：故障大百科 (Troubleshooting)

| 报错关键词 | 含义 | 解决方案 |
| :--- | :--- | :--- |
| `Permission Denied` | 串口被其他程序占用 | 关闭其他串口监控软件（如串口调试助手） |
| `Not in sync` | 选错了板卡或波特率 | 重新确认顶部工具栏的开发板型号 |
| `Compilation Failed` | 积木逻辑有空缺 | 检查是否有未填写的参数框（淡黄色圆圈） |
| `Port not found` | 物理连接断开 | 重新插拔 USB 线，或更换电脑插口 |
