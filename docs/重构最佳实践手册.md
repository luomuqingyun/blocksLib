## 技术模式

### Pattern 1: 单对象替换

**Before**:
```typescript
arduinoGenerator.definitions_['obj'] = `Type obj;`;
```

**After**:
```typescript
arduinoGenerator.addVariable('obj', `Type obj;`);
```

**适用**: 简单对象实例化 (57%的情况)

### Pattern 2: 结构体/函数

**Before**:
```typescript
arduinoGenerator.definitions_['struct'] = `
struct MyStruct {
  int field;
};
`;
```

**After**:
```typescript
arduinoGenerator.functions_['struct'] = `
struct MyStruct {
  int field;
};
`;
```

**适用**: 结构体定义、函数定义 (31%的情况)

### Pattern 3: 多变量拆分

**Before**:
```typescript
arduinoGenerator.definitions_['objs'] = `
Type1 obj1;
Type2 obj2;
`;
```

**After**:
```typescript
arduinoGenerator.addVariable('obj1', `Type1 obj1;`);
arduinoGenerator.addVariable('obj2', `Type2 obj2;`);
```

**适用**: 复杂声明优化 (12%的情况)

### Pattern 4: 领域逻辑 Hook 化 (Hook Extraction)

**场景**: 当 React Context 变得过于臃肿（超过 500 行），包含大量副作用和底层逻辑时。

**实践**: 
1. 将业务操作（如文件 IO、串口通讯协议）提取到独立的 Custom Hooks 中。
2. Context 仅保留最小状态集和 Provider 分发。

**示例**:
- `FileSystemContext` -> `useProjectOps`, `useAutoBackup`
- `SerialContext` -> `useSerialMonitor`, `useSerialActions`

**收益**: 
- **高内聚**: 逻辑逻辑按领域物理隔离。
- **易测试**: 纯逻辑 Hook 相比 Context 及其 UI 绑定更易于单元测试。
- **可复用**: 相同逻辑可在不同组件或页面中快速复用，无需依赖庞大的 Context。
