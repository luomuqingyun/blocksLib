## 1. 战略综述 (Strategic Overview)

基于“技术壁垒+用户体验”双轮驱动模型，本项目 2026 年的核心目标是：**从一个好用的工具，进化为一个繁荣的生态。**

### 三大我们要赢的战役 (Must-Win Battles)
1.  **AI 平民化 (TinyML Revolution)**: 降低 AI 门槛，从“写代码”转变为“采集数据 -> 训练 -> 部署”的一站式工作流。
2.  **插件生态化 (Ecosystem First)**: 建立开发者获利机制，充实插件库。
3.  **云端社交化 (Social Coding)**: 打通本地与云端，形成社区分享闭环。

## 2. Phase 1: Edge AI 体验重塑 (Q1)
**代号**: *"Magic Wand" (魔杖)*  
**核心目标**: 让小学生也能训练出识别手势的模型。

### 2.1 技术实施方案
-   **底层重构 (`src/modules/core/tinyml.ts`)**:
    -   废弃当前手动输入 `KNN(x, y, z)` 的低效方式。
    -   引入 **自动数据流 (Auto-Data Stream)**: 创建 `Serial Data Sampler` 积木，自动将传感器通过串口发送的数据（CSV格式）录制到内存缓冲区。
-   **新增预设模型**:
    -   集成 `TensorFlow Lite for Microcontrollers` 的 C++ 头部库（作为内置插件）。
    -   提供 `Motion Recognition` (运动识别) 和 `Voice Command` (语音指令) 两个预训练骨架。
-   **交互向导 (Wizard UI)**:
    -   开发 `TinyML Wizard` 对话框：引导用户按照 "连接 -> 摇晃录制 -> 点击训练 -> 烧录" 四步走。

### 2.2 交付物 (Deliverables)
-   [ ] 新版 `TinyML` 积木库 (支持流式采集)。
-   [ ] "Magic Wand" 官方示例项目。
-   [ ] 配套的 B 站/YouTube 演示视频脚本。

## 3. Phase 2: 插件市场基础设施 (Q2)
**代号**: *"Bazaar" (集市)*  
**核心目标**: 让第三方开发者能在 10 分钟内发布一个插件。

### 3.1 技术实施方案
-   **CLI 发布工具 (`eb-cli`)**:
    -   基于 Node.js 开发命令行工具。
    -   功能: `eb init` (初始化模板), `eb pack` (打包校验), `eb publish` (发布到 GitHub Releases)。
-   **CI/CD 流水线**:
    -   建立 GitHub Action 模板，自动检测插件的 `manifest.json` 合规性，并更新中央 `marketplace.json` 索引。
-   **沙箱性能优化**:
    -   升级 `src/sandbox`，引入 `SharedArrayBuffer` 实现主线程与沙箱的高频数据共享（为高性能游戏引擎插件铺路）。

### 3.2 交付物 (Deliverables)
-   [ ] `eb-cli` npm 包。
-   [ ] 开发者文档门户 (Developer Portal)。
-   [ ] 首批 5 个精选第三方插件（OLED驱动, 复古游戏引擎, 物联网MQTT, 等）。

---

## 4. Phase 3: 云端闭环 (Q3)
**代号**: *"Nimbus" (雨云)*  
**核心目标**: 每一个项目链接都是一个广告。

### 4.1 技术实施方案
-   **WebAssembly 播放器**:
    -   利用当前的 `sandbox` 技术，将 Blockly 运行环境剥离为一个纯前端的 Web App。
    -   支持在浏览器中“只读”查看积木逻辑和生成的 C++ 代码。
-   **项目托管服务**:
    -   后端: 使用 AWS S3 或 阿里云 OSS 存储 `.ebproj` 文件。
    -   前端: 在 TopBar 增加 "Share" 按钮，生成 `https://eb.studio/p/<uuid>` 短链接。
-   **Deep Link 唤起**:
    -   实现自定义协议 `embedblocks://open?url=...`，支持从浏览器一键拉起本地 IDE 并克隆项目。

### 4.2 交付物 (Deliverables)
-   [ ] Web 版项目查看器 (Viewer)。
-   [ ] 桌面端 "Share" 功能插件。

## 5. 资源需求与风险 (Resources & Risks)

### 5.1 人力预估
-   **Core Dev (我)**: 负责架构设计、Electron 底层及 CLI 工具。
-   **Frontend Dev**: 负责云端 Viewer 及官网建设。
-   **Content Creator**: 负责录制视频、编写教程（至关重要，技术再好也怕巷子深）。

### 5.2 风险管理
-   **风险**: 硬件碎片化导致 AI 库兼容性差。
    -   **对策**: Q1 阶段仅官方支持 `ESP32-S3` 和 `Arduino Nano 33 BLE` 两款主流 AI 板卡，确保体验收敛，后续再通过社区扩展。
-   **风险**: 插件质量参差不齐。
    -   **对策**: 建立“官方认证 (Verified)”机制，经过人工 Code Review 的插件才给予金标认证。
