### [16:50] 代码编辑区“滚动跳转”现象深度诊断 (Editor Scrolling & Jump Analysis)

#### 1. 现象描述 (Phenomenon)
用户在左侧积木区添加/修改积木时，右侧 Monaco Editor 的滚动条偶尔会发生意外跳转（如回弹到顶部或偏移几行），即便编辑器窗口尺寸并未发生改变。

#### 2. 核心机理 (Root Causes)
- **全量更新开销 (Virtual DOM Sync)**: 
    - 积木变更触发 `AppInner` 渲染，通过 `setValue` 全量重写编辑器内容。
    - Monaco 在接收到新值时，若行数发生变动，会重新计算虚拟滚动高度。在 React 的 Reconciliation（调和）期间，如果布局发生微秒级的失稳，编辑器会执行一次“纠偏”。
- **布局敏感度 (Automatic Layout)**:
    - 开启 `automaticLayout: true` 后，编辑器会高频检测父容器尺寸。
    - 在 React 组件重绘瞬间，父容器可能会产生极短暂的“状态计算期”，导致 Monaco 误判定为尺寸变化并强制触发 `layout()`。
- **边界锚点误差 (Scroll Beyond Last Line)**:
    - 若此项设为 `true`，编辑器底部会生成动态空白区。
    - 在内容大幅变化时，滚动条的比例算法会因为“溢出空白”的重新计算而产生坐标映射误差，导致跳转。

#### 3. 应对措施 (Mitigation)
- **已实施**: 
    - 稳定了 `onCodeChange` 的函数引用（`useRef`），减少不必要的重组件调和。
    - 设置 `scrollBeyondLastLine: false` 以确定物理边界。
- **未来路线**: 考虑从 `setValue` 迁移为基于 `pushEditOperations` 的 Diff 增量更新，从根本上保留编辑器视图状态。

### STM32 Local Patch 编译链修复 (Local Patch Build Chain Fix)

  - Flash 代码映射: `4/6/8` → `x8`, `B/C` → `xB`, `D/E` → `xE`, `F/G/I/Z` → `xG`
