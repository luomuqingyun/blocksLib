# 扩展市场使用与开发指南 (Extension Marketplace Guide)

> [!NOTE]
> 本文档分为 **用户篇** (如何使用) 和 **开发者篇** (如何发布)。

---

## 🟢 用户篇：如何安装扩展 (For Users)

EmbedBlocks 采用去中心化的市场设计，您可以添加任意第三方维护的“软件源”来获取插件。

### 1. 添加市场源
1.  打开 IDE 顶部的 **扩展中心 (Extensions)** 图标。
2.  切换到 **插件市场 (Marketplace)** 选项卡。
3.  点击右上角的 **添加市场源 (+)** 按钮。
4.  输入市场源地址并确认。

### 2. 常见市场源格式
系统支持两种链接格式：
*   **GitHub 仓库主页** (推荐): 
    *   例如: `https://github.com/username/my-blocks-lib`
    *   系统会自动查找该仓库 `main` 分支根目录下的 `marketplace.json`。
*   **JSON 直链**: 
    *   例如: `https://mysite.com/extensions/market.json`
    *   必须以 `http://` 或 `https://` 开头。

### 3. 故障排查 (Troubleshooting)

#### 🔴 现象：添加源后列表为空 (No extensions found)
如果您看到 "未找到任何扩展" 的提示，通常只有以下几个原因：

1.  **链接无效 (404 Error)**: 
    *   **最常见原因**。您提供的链接无法访问。
    *   **自测方法**: 将您输入的 URL (如果是 GitHub 仓库，请拼写为 `raw.githubusercontent.com/.../marketplace.json`) 复制到浏览器地址栏。如果浏览器显示 `404 Not Found`，IDE 自然也无法读取。
2.  **网络问题**: 
    *   GitHub Raw 域名在国内可能访问不稳定。
3.  **JSON 格式错误**:
    *   如果文件存在但格式非法（如多余的逗号），解析会失败。

## 🔵 开发者篇：如何发布扩展 (For Developers)

您只需要一个静态文件服务器（如 GitHub Pages 或 GitHub Repo）即可建立自己的插件市场。

### 1. 准备工作
创建一个 GitHub 仓库（例如 `my-blocks-lib`），并在根目录下创建两个核心文件：
1.  `marketplace.json`: 市场的索引文件。
2.  `plugins/`: 存放插件 ZIP 包的文件夹（或者使用 GitHub Releases 托管 ZIP）。

### 2. marketplace.json 规范
这是市场的核心索引，请严格遵守以下格式：

```json
{
  "name": "我的插件库",
  "extension": [  // 注意：某些旧版本可能使用 "extensions"，请以最新版为准，目前系统兼容两者
    {
      "id": "author.my-plugin",   // [必须] 唯一标识符
      "version": "1.0.0",         // [必须] 版本号
      "name": "超级传感器",        // [显示] 插件名称
      "description": "这是一个很棒的插件",
      "author": "MyName",
      "contributes": {             // [重要] 如果是硬件板卡插件，必须包含此项
        "boards": ["board.json"]
      },
      "icon": "https://...",      // [可选] 图标 URL (64x64 PNG)
      "downloadUrl": "https://github.com/username/repo/releases/download/v1.0/plugin.zip" // [必须] 下载直链
    }
  ]
}
```

### 3. 插件包 (ZIP) 规范
`downloadUrl` 指向的 ZIP 文件解压后，**根目录必须直接包含** `manifest.json`。

**❌ 错误结构**:
```text
my-plugin.zip
└── my-plugin/      <-- 多了一层文件夹
    ├── manifest.json
    └── ...
```

**✅ 正确结构**:
```text
my-plugin.zip
├── manifest.json
├── blocks.js
└── ...
```

### 4. 发布流程
1.  开发并测试您的插件。
2.  将插件文件打包为 `.zip`。
3.  将 ZIP 上传到 GitHub Releases 或仓库中。
4.  获取 ZIP 的 **下载直链**。
5.  更新 `marketplace.json` 中的 `version` 和 `downloadUrl`。
6.  提交代码 (`git push`)。

用户只需点击 IDE 中的“刷新”按钮，即可看到最新版本。
