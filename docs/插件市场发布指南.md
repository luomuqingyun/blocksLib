# 扩展市场发布指南 (Marketplace Publishing Guide)

> [!IMPORTANT]
> **适用对象**：第三方插件开发者（User-Level Plugins）。
> **与内部开发的区别**：本文档描述的是如何发布**无需编译软件**即可分发的独立插件。如果您是希望直接修改 EmbedBlocks 源码，请参考 [内部开发架构指南](./核心开发指南.md)。

---

## 1. 核心机制：去中心化分发

EmbedBlocks 扩展市场不依赖于单一的中央服务器。它通过读取开发者提供的 `marketplace.json` 清单文件来发现插件。这种模式具有以下优势：
- **零成本**：开发者无需搭建服务器，利用 GitHub Pages 或 Release 即可发布。
- **高自由度**：任何个人或组织都可以建立自己的私有或公开市场。
- **实时性**：更新 JSON 文件后，所有订阅该源的用户都能立即看到新版本。

---

## 2. 准备发布包

在将插件列入清单之前，你需要准备好物理包：

1.  **代码清理**：
    - 删除 `node_modules`、`.pio`、`.git` 等开发过程产生的多余文件夹。
    - 确保 C++ 库文件放在 `libraries/` 目录下。
2.  **校验 Manifest**：
    - 确保根目录下的 `manifest.json` 配置了正确的 `id`、`version` 和 `entry`。
    - **注意**：插件内的 `id` 必须是全局唯一的（建议使用 `author.plugin_name` 格式）。
3.  **压缩打包**：
    - 将插件根目录下的所有内容直接压缩为 `.zip`。
    - **推荐方式**：在 GitHub 上发布 Release，直接使用自动生成的 `Source code (zip)` 链接。

---

## 3. 设计 marketplace.json

这个文件是市场的“灵魂”，它定义了用户在 UI 中看到的预览信息。

### 3.1 文件架构

```json
{
  "name": "官方插件库",           // 市场名称（暂未使用，建议保留）
  "description": "...",         // 市场描述
  "extensions": [
    {
      "id": "oled-u8g2",         // [必须] 必须与插件内部 manifest.json 的 id 相同
      "version": "1.2.0",        // [必须] 必须与内部版本号相同，用于版本比对
      "name": "OLED U8g2",        // [必须] 在市场列表中显示的友好名称
      "description": "...",      // [推荐] 功能介绍，支持简单换行
      "author": "EmbedBlocks",   // [推荐] 作者姓名或组织
      "icon": "https://...",     // [推荐] 64x64 的图片直链，支持 PNG/SVG
      "downloadUrl": "https://..."// [必须] ZIP 文件的 HTTPS 直链
    }
  ]
}
```

### 3.2 关键注意事项
-   **HTTPS 强制**：出于安全考虑，IDE 拒绝从 `http` 协议下载任何内容。
-   **GitHub 链接支持**：如果你将 `marketplace.json` 放在 GitHub 仓库根目录，用户只需添加仓库主页链接（如 `https://github.com/user/repo`），系统会自动将其归一化为 raw 链接。
-   **版本更新**：当你想发布新版本时，只需先上传新的 ZIP，然后更新此 JSON 中的 `version` 字段，用户端会自动标记为“可安装新版本”。

---

## 4. 发布与分发

1.  **上传文件**：将你的插件 ZIP 上传到可公开访问的静态空间。
2.  **更新清单**：编辑并上传你的 `marketplace.json`。
3.  **分享给用户**：
    -   用户打开 **扩展中心 (Extensions)**。
    -   点击 **插件市场 (Marketplace)** 选项卡。
    -   点击 **添加市场源 (+)**。
    -   输入你的仓库链接或 JSON 直链并确认。

---

## 5. 故障排查 (Troubleshooting)

-   **插件不显示**：
    -   检查 `marketplace.json` 格式是否符合标准 JSON 规范。
    -   检查 `extensions` 是否为数组。
-   **安装失败**：
    -   如果是 GitHub 链接，检查仓库是否为私有（必须公开）。
    -   检查 `downloadUrl` 是否可以直接在浏览器中下载。
-   **安装后无积木**：
    -   检查 ZIP 内部结构。ZIP 解压后应直接看到 `manifest.json`，而不应多出一层嵌套文件夹。
