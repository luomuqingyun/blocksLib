# STM32 芯片支持策略说明 (STM32 Support Policy)

本文档记录了 EmbedBlocks Studio 对 STM32 芯片的支持策略。我们目前集成了 **1,430** 款 STM32 芯片的数据。

---

> [!TIP]
> 这是一个基于本地 PlatformIO 软件包（framework-arduinoststm32）自动同步的系统。运行以下指令可刷新最新的支持状态：
> ```bash
> npm run gen:stm32
> ```
> ```
> 指令会自动检查 PlatformIO 包更新，随后运行官方变体解析、引脚扫描和布局合并，生成最终的 Board JSON。

## 📊 支持标准

为了确保开发体验的绝对稳定性，我们对芯片支持采取了**严格过滤策略**：

1. **唯一来源**: 仅包含在本地 PlatformIO 安装的 `framework-arduinoststm32` 包中拥有明确 `Variant` 定义的型号。
2. **可靠编译**: 每一款被包含在内的芯片都关联了正确的编译参数（MCU, Platform, Variant），确保用户点击“编译”即可通过，无需手动配置。
3. **精准指引**: 我们提供了针对这 **1,430** 款芯片的精确引脚映射与 SVG 视觉预览。

---

## 📝 暂不支持的具体型号清单

以下系列或型号虽然在底层引脚定义中存在，但由于尚未完成 Arduino 官方核心的适配，目前暂未包含在 EmbedBlocks 中：

### 1. STM32H7R/S 系列 (早期预览)
- STM32H7R3... (全系列)
- STM32H7R7... (全系列)
- STM32H7S3... (全系列)
- STM32H7S7... (全系列)

### 2. STM32N6 系列 (AI 增强型)
- STM32N645... (全系列)
- STM32N647... (全系列)
- STM32N655... (全系列)
- STM32N657... (全系列)

### 3. STM32L1 系列 (特殊后缀项)
- STM32L100C6-A, STM32L100R8-A...
- STM32L151...-A / -X 系列
- STM32L152...-A / -X 系列
- STM32L162...-A / -X 系列
*(注：标准 STM32L1xx 型号已 100% 支持)*

### 4. STM32WBA 系列 (低功耗蓝牙)
- STM32WBA50... (**已支持**, 通过 `fix_wba_build.py` 拦截脚本解决 PlatformIO 识别冲突)
- STM32WBA52 / WBA54 / WBA55 (部分尚未在官方库注册的型号，暂不推荐，但在 WBA50 框架下可尝试适配)

---

## 🛠️ 后续计划

1. **自动同步**: 随着 PlatformIO 框架的定期更新（脚本已内置 `pio pkg update`），一旦本地包增加了新系列的定义，运行 `npm run gen:stm32` 即可自动解锁对应型号。
2. **社区反馈**: 如果您发现某款芯片官方已支持但 EmbedBlocks 仍未列出，请通过 GitHub Issue 告知，我们会在下一个版本中修正过滤策略。

> [!IMPORTANT]
> 我们的原则是：**不提供无法稳定编译的预览版硬件。** 您在 EmbedBlocks 列表中看到的 1,430 款芯片，均可直接投入生产级开发。
