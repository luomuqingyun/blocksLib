# 未支持芯片型号说明 (Unsupported STM32 Chips)

本文档记录了目前 EmbedBlocks Studio 尚未支持的 STM32 芯片型号（共计 **73** 款）。这些型号虽然存在于 ST 官方的 Open Pin Data 中，但由于底层 **Arduino Core STM32** 尚未提供对应的 Variant 定义，目前无法在软件中直接使用。

---

> [!TIP]
> 这是一个静态文档。如需查看**实时**的未支持芯片统计和列表，请运行：
> ```bash
> npm run sync:stm32
> ```
> 脚本会在执行最后自动输出最新的 "Unsupported Chips Report"。

## 📊 统计概览

| 芯片系列 | 未支持数量 | 主要原因 |
| :--- | :--- | :--- |
| **STM32H7 (R/S)** | 13 | 2024/2025 年发布的超高性能新系列，Arduino 核心尚未适配。 |
| **STM32N6** | 24 | ST 首款带 NPU 的工业级 AI 芯片，架构较新，暂无 Variant。 |
| **STM32L1 (-A)** | 25 | 特殊后缀 (-A/-X) 型号在 Open Data 中定义独立，但 Arduino 核心通常将其合并在通用型号中或暂未收录。 |
| **STM32WBA** | 11 | 低功耗蓝牙新系列，目前仅部分型号完成适配。 |
| **总计** | **73** | |

---

## 📝 具体型号清单

### 1. STM32H7R/S 系列 (高性能)
- STM32H7R3... (全系列)
- STM32H7R7... (全系列)
- STM32H7S3... (全系列)
- STM32H7S7... (全系列)

### 2. STM32N6 系列 (AI 增强型)
- STM32N645... (全系列)
- STM32N647... (全系列)
- STM32N655... (全系列)
- STM32N657... (全系列)

### 3. STM32L1 系列 (特殊后缀项)
- STM32L100C6-A, STM32L100R8-A...
- STM32L151...-A / -X 系列
- STM32L152...-A / -X 系列
- STM32L162...-A / -X 系列
*(注：普通 STM32L1xx 型号已 100% 支持)*

### 4. STM32WBA 系列 (低功耗蓝牙)
- STM32WBA50...
- STM32WBA52... (部分)
- 以及其他尚未在 `Arduino_Core_STM32` 注册的 WBA 型号。

---

## 🛠️ 后续计划

1. **跟进官方更新**: 随着 `third_party/Arduino_Core_STM32` 子模块的定期同步，一旦上游仓库增加了相应的 Variant 定义，运行 `npm run sync:stm32` 即可自动解锁对应型号。
2. **手动映射适配**: 对于某些急需且 Arduino 核心已支持但名称不匹配的型号，可在 `stm32duino_support_registry.json` 中添加手动映射规则。

> [!NOTE]
> 我们专注于维持核心稳定性。对于目前支持的 **1,430** 款芯片，我们已确保其引脚定义与代码生成的 100% 可靠性。
