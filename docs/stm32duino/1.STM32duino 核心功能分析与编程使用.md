# STM32duino 核心功能分析与编程使用

## 一、项目概述

STM32duino (Arduino_Core_STM32) 是一个将 STM32 微控制器集成到 Arduino 开发环境的开源项目,让开发者能够使用 Arduino API 来开发 STM32 应用程序。

### 核心架构基础

项目基于以下三个主要组件构建:

1. **STM32Cube MCU Packages**
   - HAL (Hardware Abstraction Layer): 标准化 API 调用,实现不同 STM32 设备间的可移植性
   - LL (Low-Layer) APIs: 轻量级、优化的专家级 API,注重性能和运行效率
   - CMSIS 设备定义

2. **CMSIS (Cortex Microcontroller Software Interface Standard)**
   - 为 Cortex-M 处理器系列提供独立于供应商的硬件抽象层

3. **GNU Arm Embedded Toolchain**
   - ARM 嵌入式 GCC 编译器和相关工具链

## 二、支持的硬件平台

STM32duino 支持超过 **500+ 种开发板**,涵盖多个 STM32 系列:

### 主流系列支持
- **STM32F 系列**: F0, F1, F2, F3, F4, F7 (经典 Cortex-M 系列)
- **STM32G 系列**: G0, G4 (主流通用型)
- **STM32H 系列**: H5, H7 (高性能系列)
- **STM32L 系列**: L0, L1, L4, L5 (低功耗系列)
- **STM32U 系列**: U0, U3, U5 (超低功耗)
- **STM32C 系列**: C0 (入门级)
- **STM32WB/WBA/WL 系列**: 无线连接系列
- **STM32MP1**: 多处理器系列(协处理器模式)

### 开发板类型
- **官方开发板**: Nucleo (32/64/144 引脚), Discovery, Eval 板
- **通用开发板**: Blue Pill, Black Pill, WeAct 系列等
- **专业板卡**: 3D 打印机主板、飞控板、物联网板等

## 三、核心功能实现

### 3.1 Arduino API 兼容层

#### 数字 I/O
```cpp
// 基本数字 I/O
pinMode(pin, mode);           // 支持 INPUT, OUTPUT, INPUT_PULLUP, INPUT_PULLDOWN
digitalWrite(pin, value);     // HIGH/LOW
int value = digitalRead(pin); // 读取数字值
```

#### 模拟功能
```cpp
// ADC 读取
int value = analogRead(pin);  // 默认 10-bit 分辨率

// DAC/PWM 输出 - 智能选择
analogWrite(pin, value);      // 自动选择最佳方式:
                              // - DAC 引脚: 真正的模拟输出
                              // - 定时器引脚: PWM 输出
                              // - 普通引脚: GPIO 翻转

// 设置 PWM 频率 (core > 1.5.0)
analogWriteFrequency(freq);   // 默认 1000Hz
```

**重要特性**: 
- 不同引脚根据硬件能力自动选择最优输出方式
- 同一定时器的所有通道共享频率设置

### 3.2 通信接口

#### Serial (UART/USART)
```cpp
// 标准 Arduino Serial API
Serial.begin(115200);
Serial.print("Hello");
Serial.println(data);

// 支持多个硬件串口
Serial1.begin(9600);   // USART1
Serial2.begin(115200); // USART2
```

#### SPI
```cpp
#include <SPI.h>

SPI.begin();
SPI.transfer(data);

// STM32 扩展功能: 硬件 CS 管理
SPIClass mySPI(MOSI_PIN, MISO_PIN, SCK_PIN, SS_PIN);
// 如果配置了 SS_PIN,芯片选择由 SPI 外设自动管理
```

**STM32 特色功能**:
- 可选择软件或硬件 CS 管理
- 支持 `SPI_TRANSMITONLY` 模式(只发送,跳过接收以提高效率)

#### I2C (Wire)
```cpp
#include <Wire.h>

Wire.begin();              // 主模式
Wire.beginTransmission(addr);
Wire.write(data);
Wire.endTransmission();
```

**高级特性**:
- **主模式**: RX/TX 缓冲区自动动态增长(独立管理,最大 255 字节)
- **从模式**: 可通过 `I2C_TXRX_BUFFER_SIZE` 静态配置缓冲区大小
- 默认缓冲区: 32 字节(兼容 Arduino)

### 3.3 HardwareTimer 库

这是 STM32duino 最强大的特色功能之一,提供对硬件定时器的完整访问。

#### 基础配置
```cpp
HardwareTimer *MyTim = new HardwareTimer(TIM1);

// 设置溢出频率 (Hz)
MyTim->setOverflow(1000, HERTZ_FORMAT); // 1kHz

// 设置溢出时间 (微秒)
MyTim->setOverflow(1000, MICROSEC_FORMAT);

// 启动定时器
MyTim->resume();
```

#### PWM 配置
```cpp
// 简化的一体化 PWM 配置
MyTim->setPWM(channel, pin, frequency, dutycycle);
// 例: 5Hz, 10% 占空比
MyTim->setPWM(1, PA8, 5, 10);

// 手动配置
MyTim->setMode(channel, TIMER_OUTPUT_COMPARE_PWM1, pin);
MyTim->setCaptureCompare(channel, dutycycle);
```

#### 回调函数
```cpp
void callback() {
  digitalWrite(LED_BUILTIN, !digitalRead(LED_BUILTIN));
}

MyTim->attachInterrupt(callback);
MyTim->resume();
```

#### 输入捕获
```cpp
// 测量外部信号频率
MyTim->setMode(channel, TIMER_INPUT_CAPTURE_RISING, pin);
MyTim->attachInterrupt(channel, IC_callback);

void IC_callback() {
  uint32_t captured = MyTim->getCaptureCompare(channel);
  // 计算频率
}
```

**关键概念**:
- 同一定时器的所有通道共享计数器,因此频率/周期相同
- 不同定时器有不同特性(如 TIM6/7 无输出引脚,用于 Tone/Servo)

### 3.4 中断管理

```cpp
// 外部中断
attachInterrupt(digitalPinToInterrupt(pin), ISR, mode);
// mode: LOW, CHANGE, RISING, FALLING, HIGH

detachInterrupt(digitalPinToInterrupt(pin));
```

### 3.5 EEPROM 模拟

STM32 无真实 EEPROM,通过 Flash 模拟实现。

```cpp
#include <EEPROM.h>

// 标准 API
EEPROM.write(address, value);
byte value = EEPROM.read(address);
EEPROM.update(address, value);  // 只在值改变时写入

// 缓冲 API (减少 Flash 擦写次数)
eeprom_buffer_fill();           // 从 Flash 加载到 RAM
eeprom_buffered_write_byte(pos, value);
eeprom_buffer_flush();          // 一次性写回 Flash
```

**注意事项**:
- Flash 写入需要擦除整个扇区/页,可能很慢
- 使用缓冲 API 可显著减少擦写次数,延长 Flash 寿命

### 3.6 CMSIS DSP 库

提供丰富的数字信号处理功能:

```cpp
// 可用的 DSP 函数类别:
// - 基本数学函数
// - 快速数学函数
// - 复数数学函数
// - 滤波器函数
// - 矩阵函数
// - 变换函数 (FFT 等)
// - 电机控制函数
// - 统计函数
// - 支持函数
```

支持多种数据类型: 8-bit, 16-bit, 32-bit 整数和 32-bit 浮点数。

## 四、高级定制功能

### 4.1 HAL 模块配置

#### 禁用未使用的模块
通过 `build_opt.h` 或 `hal_conf_extra.h` 禁用模块可显著减少 Flash/RAM 占用:

```cpp
// 在 build_opt.h 中
-DHAL_ADC_MODULE_DISABLED
-DHAL_DAC_MODULE_DISABLED
-DHAL_I2C_MODULE_DISABLED
// ... 其他模块
```

可禁用的模块包括:
ADC, CAN, DAC, ETH, I2C, RTC, SD, SPI, TIM, UART, USB, QSPI 等

#### 仅使用 HAL 模块而不被 Arduino API 占用
```cpp
// 例如完全控制 I2C
#define HAL_I2C_MODULE_ONLY
// 不包含 Wire.h,直接使用 HAL I2C 函数
```

### 4.2 核心回调功能

启用核心回调:
```cpp
// 在 build_opt.h 中
-DCORE_CALLBACK
```

### 4.3 自定义构建选项

通过 `build_opt.h` 文件自定义编译:
```cpp
// 优化级别
-Os                          // 优化大小
-O3                          // 优化速度

// 启用浮点打印/扫描
-u _printf_float
-u _scanf_float

// 启用 LTO (链接时优化)
-flto

// 启用调试符号
-g

// 自定义定义
-DUSER_CUSTOM_DEFINE
```

### 4.4 振荡器配置

可重新定义的振荡器值(在 `hal_conf_extra.h` 中):
```cpp
#define HSE_VALUE    8000000U   // 外部高速晶振
#define HSI_VALUE    16000000U  // 内部高速晶振
#define LSE_VALUE    32768U     // 外部低速晶振
#define LSI_VALUE    32000U     // 内部低速晶振
```

## 五、编程使用示例

### 5.1 基础 Blink 程序
```cpp
void setup() {
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);
  delay(1000);
  digitalWrite(LED_BUILTIN, LOW);
  delay(1000);
}
```

### 5.2 硬件定时器中断
```cpp
HardwareTimer *MyTim;

void setup() {
  pinMode(LED_BUILTIN, OUTPUT);
  
  MyTim = new HardwareTimer(TIM1);
  MyTim->setOverflow(2, HERTZ_FORMAT);  // 2Hz = 0.5秒周期
  MyTim->attachInterrupt(toggleLED);
  MyTim->resume();
}

void toggleLED() {
  digitalWrite(LED_BUILTIN, !digitalRead(LED_BUILTIN));
}

void loop() {
  // 主循环空闲,LED 由定时器中断控制
}
```

### 5.3 PWM 控制
```cpp
HardwareTimer *pwmTimer;

void setup() {
  pwmTimer = new HardwareTimer(TIM2);
  
  // 配置 PWM: 1kHz, 50% 占空比
  pwmTimer->setPWM(1, PA0, 1000, 50);
}

void loop() {
  // 动态调整占空比
  for(int duty = 0; duty <= 100; duty++) {
    pwmTimer->setCaptureCompare(1, duty, PERCENT_COMPARE_FORMAT);
    delay(50);
  }
}
```

### 5.4 UART 通信
```cpp
void setup() {
  Serial.begin(115200);
  Serial1.begin(9600);  // 第二个串口
}

void loop() {
  if(Serial.available()) {
    char c = Serial.read();
    Serial1.write(c);  // 转发到另一个串口
  }
  
  if(Serial1.available()) {
    String data = Serial1.readString();
    Serial.println(data);
  }
}
```

### 5.5 I2C 传感器读取
```cpp
#include <Wire.h>

#define SENSOR_ADDR 0x68

void setup() {
  Serial.begin(115200);
  Wire.begin();
}

void loop() {
  Wire.beginTransmission(SENSOR_ADDR);
  Wire.write(0x00);  // 寄存器地址
  Wire.endTransmission();
  
  Wire.requestFrom(SENSOR_ADDR, 2);
  if(Wire.available() >= 2) {
    uint8_t high = Wire.read();
    uint8_t low = Wire.read();
    int16_t value = (high << 8) | low;
    Serial.println(value);
  }
  
  delay(100);
}
```

## 七、上传方式

STM32duino 支持多种程序上传方法:

1. **ST-Link**: 官方调试器(最常用)
2. **DFU (USB)**: 通过 USB 直接烧录
3. **Serial (UART)**: 通过串口引导加载程序
4. **HID Bootloader**: USB HID 引导加载程序
5. **Mass Storage**: 某些板支持拖拽上传

## 八、调试功能

### 支持的调试方式
- **ST-Link**: 通过 OpenOCD
- **Serial Wire Debug (SWD)**: 硬件调试
- **Printf 调试**: 通过串口输出调试信息
- **断言检查**: 启用 `USE_FULL_ASSERT` 进行运行时检查

### 断言示例
```cpp
// 在 build_opt.h 中启用

// 自定义断言失败处理

  Serial.print(" (");

  Serial.println(")");

}
```

## 九、与标准 Arduino 的区别

### 优势
1. **更强大的硬件**: 32-bit ARM Cortex-M,更高时钟频率
2. **丰富的外设**: 多个 UART/SPI/I2C,高级定时器,DAC 等
3. **直接访问 HAL**: 可突破 Arduino API 限制使用完整硬件功能
4. **低功耗选项**: L/U 系列提供超低功耗模式
5. **性价比**: STM32 芯片通常比同性能 Arduino 板更便宜

### 注意事项
1. **引脚命名**: 使用 PA0, PB1 等而非仅数字编号
2. **电压**: 多数 STM32 为 3.3V,需要电平转换与 5V 设备通信
3. **Flash 生命周期**: EEPROM 模拟会消耗 Flash 擦写次数
4. **定时器复杂性**: 需要理解定时器架构才能充分利用

## 十、总结

STM32duino 通过以下方式成功将专业级 STM32 微控制器引入 Arduino 生态:

### 核心成就
1. **保持兼容性**: 大部分 Arduino 代码可直接运行
2. **扩展功能**: 提供访问 STM32 高级特性的接口(HardwareTimer, HAL 配置等)
3. **灵活性**: 支持从简单 Blink 到复杂 DSP 处理的各种应用
4. **丰富支持**: 500+ 开发板,完整的库生态

### 适用场景
- **学习嵌入式开发**: Arduino 简易性 + STM32 强大功能
- **快速原型开发**: 利用 Arduino 库快速实现功能
- **产品开发**: 可逐步从 Arduino API 迁移到纯 HAL 以优化性能
- **教育培训**: 平滑从 Arduino 过渡到专业嵌入式开发

STM32duino 是连接 Arduino 简易性和 STM32 专业性的完美桥梁!
