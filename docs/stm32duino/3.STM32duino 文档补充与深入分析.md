## 二、重要补充内容

### 2.1 ADC 高级功能

#### ADC 内部通道读取
核心版本 1.5.0+ 支持读取内部通道：

```cpp
// 读取内部温度传感器
int temp = analogRead(ATEMP);

// 读取内部参考电压
int vref = analogRead(AVREF);

// 读取电池电压
int vbat = analogRead(AVBAT);
```

#### ADC 分辨率配置
```cpp
// 设置 ADC 分辨率 (默认 10-bit)
analogReadResolution(12);  // 12-bit: 0-4095

// 设置 PWM/DAC 输出分辨率
analogWriteResolution(12); // 12-bit
```

#### ADC 采样时间配置
可通过 `build_opt.h` 自定义：
```cpp
-DADC_SAMPLINGTIME=ADC_SAMPLETIME_28CYCLES
-DADC_CLOCK_DIV=ADC_CLOCK_SYNC_PCLK_DIV4
```

### 2.2 DAC 输出缓冲控制

自 Core 2.8.0 版本起，可禁用 DAC 输出缓冲：

```cpp
// 在 build_opt.h 中
-DDISABLE_DAC_OUTPUTBUFFER
```

### 2.3 HardwareSerial 高级配置

#### 启用多个硬件串口
```cpp
// 方法 1: 在代码中声明
HardwareSerial Serial2(PA3, PA2);  // RX, TX

// 方法 2: 在 build_opt.h 中
-DENABLE_HWSERIAL2
-DPIN_SERIAL2_RX=PA3
-DPIN_SERIAL2_TX=PA2
```

#### LPUART 支持
低功耗串口使用 `SerialLP1`：
```cpp
SerialLP1.begin(9600);  // LPUART1
```

#### serialEvent 回调
只有通过 `ENABLE_HWSERIALx` 定义的串口才支持 `serialEventx()` 回调。

### 2.4 noinit 变量（跨重置保持数据）

自 Core 1.9.0 版本支持，变量在软复位时保持数据：

```cpp
// 标记变量为 noinit
unsigned boot_count __attribute__((__section__(".noinit")));

void setup() {
  Serial.begin(115200);
  
  // 检查是否为上电复位
  if (__HAL_RCC_GET_FLAG(RCC_FLAG_BORRST)) {
    boot_count = 1;  // 首次上电，初始化
  }
  __HAL_RCC_CLEAR_RESET_FLAGS();
  
  Serial.print("Boot count: ");
  Serial.println(boot_count);
  ++boot_count;
}

void loop() {}
```

### 2.5 Servo 库实现细节

**关键点（文档未详细说明）：**

1. **任意 GPIO 支持**：使用 HardwareTimer 作为时间基准，不是 PWM 输出
2. **最多 12 个舵机**：使用单个定时器实例 `TIMER_SERVO`
3. **软件 PWM 生成**：在中断中顺序生成各舵机信号

```
时序示意：
Servo1: __|¯¯|__________________________________|¯¯|__
Servo2: _____|¯¯|__________________________________|¯¯
```

**性能优化提示：**
```cpp
// 使用硬件 PWM (仅限定时器输出引脚)
HardwareTimer *pwm = new HardwareTimer(TIM2);
pwm->setPWM(1, PA0, 50, 7.5);  // 50Hz, 7.5% = 1.5ms
```

## 三、文档未涵盖的重要库

### 3.1 STM32FreeRTOS

提供 FreeRTOS v9.x 和 v10.x 的 STM32 移植版本。

```cpp
#include <STM32FreeRTOS.h>

void TaskBlink(void *pvParameters) {
  pinMode(LED_BUILTIN, OUTPUT);
  for (;;) {
    digitalWrite(LED_BUILTIN, !digitalRead(LED_BUILTIN));
    vTaskDelay(pdMS_TO_TICKS(1000));
  }
}

void setup() {
  xTaskCreate(TaskBlink, "Blink", 128, NULL, 1, NULL);
  vTaskStartScheduler();
}

void loop() {}  // 不会执行
```

**配置文件：**
- `STM32FreeRTOSConfig.h` - 用户自定义配置
- `STM32FreeRTOSConfig_extra.h` - 额外配置

**内存分配方案：**
- `heap_1.c` 到 `heap_5.c`
- `heap_useNewlib_ST.c` (默认，线程安全)

### 3.2 STM32duinoBLE

蓝牙低功耗支持，适用于 STM32WB 系列。

```cpp
#include <STM32duinoBLE.h>

BLEService ledService("19B10000-E8F2-537E-4F6C-D104768A1214");
BLEByteCharacteristic switchCharacteristic("19B10001-E8F2-537E-4F6C-D104768A1214", 
                                           BLERead | BLEWrite);

void setup() {
  BLE.begin();
  BLE.setLocalName("STM32 LED");
  BLE.setAdvertisedService(ledService);
  ledService.addCharacteristic(switchCharacteristic);
  BLE.addService(ledService);
  BLE.advertise();
}

void loop() {
  BLE.poll();
  if (switchCharacteristic.written()) {
    digitalWrite(LED_BUILTIN, switchCharacteristic.value());
  }
}
```

### 3.4 Tone 库

内置 Tone 功能用于生成方波音调：

```cpp
void setup() {
  // 在 PA0 上生成 1000Hz 音调
  tone(PA0, 1000);
  delay(500);
  noTone(PA0);
  
  // 指定持续时间
  tone(PA0, 2000, 1000);  // 2000Hz, 1秒
}
```

**注意：** 使用 `TIMER_TONE` 定时器。

### 3.5 USB 功能扩展

#### USB CDC (虚拟串口)
某些板卡支持 USB CDC，可通过 `Serial` 或 `SerialUSB` 访问。

#### USB MSC (大容量存储)
支持将 STM32 作为 U 盘使用。

## 五、核心回调机制

```cpp

```

```cpp

void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
  // 自定义处理
}

void HAL_ADC_ConvCpltCallback(ADC_HandleTypeDef* hadc) {
  // 自定义处理
}
```

## 七、调试与故障排除补充

### 7.1 Printf 输出重定向

```cpp
// 重定向 printf 到 Serial
extern "C" int _write(int file, char *ptr, int len) {
  (void)file;
  Serial.write(ptr, len);
  return len;
}
```

### 7.2 OpenOCD 调试配置

支持通过 ST-Link 进行硬件调试，参见 Wiki 的 "How to debug" 页面。

### 7.3 常见问题

1. **Flash 写入缓慢**：EEPROM 操作可能耗时 20-200ms
2. **定时器冲突**：Tone/Servo/PWM 可能争用定时器资源
3. **USB 缓冲区**：USB CDC 可能需要调整缓冲区大小
