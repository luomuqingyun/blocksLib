# Modification Memo - 2026-01-08

## Overview: I18n First & Structural Optimization

This update ensures all internal development components (examples, custom hardware) comply with the "I18n First" standard and implements auto-discovery to reduce developer workload.

### 1. Internationalization (I18n) Improvements

- **Custom Hardware Dual-Mode I18n**: 
  - Updated `Board`, `BoardSeries`, and `BoardFamily` types to support `I18nString` (`{zh: string; en: string}`).
  - Refactored all custom boards in `src/config/custom/` to use localized names and pin labels.
  - Updated `NewProjectModal` and `TopBar` to dynamically render localized board names.
- **Example Modules I18n**:
  - Replaced hardcoded strings in `basic_led`, `advanced_dht`, and `complex_module` with `Blockly.Msg` references.
  - Added module name translation keys to `zh.json` and `en.json`.
  - Populated `setupBlocklyLocales.ts` with bilingual definitions for example-specific blocks.

### 2. Structural & Workflow Optimization

- **Zero-Config Auto-Discovery**:
  - **Modules**: Refactored `src/modules/index.ts` to automatically scan and register any module in `src/modules/extensions/` using `import.meta.glob`.
  - **Hardware**: Refactored `src/config/all_boards.ts` to automatically discover and group custom boards from `src/config/custom/` into a new **"Chip Lab" (芯片实验室)** family.
- **Documentation Overhaul**:
  - Updated `docs/核心开发指南.md` with a "Source Code Geography" guide to clarify folder roles.
  - Added a comprehensive project directory tree to the guide for easier onboarding.
  - Updated FAQ and setup instructions to reflect the new automated workflow.

### 3. Verification & Stability

- Fixed lint errors regarding duplicate locale property names in `setupBlocklyLocales.ts`.
- Verified that localized contents appear correctly in the UI upon language switch.
- Confirmed that new extensions/boards are detected automatically without manual registry edits.
