## [1.2.4] - 2026-02-07

#### STM32 数据源重构
- **单一可信源**: 彻底移除了 `third_party/Arduino_Core_STM32` 子模块依赖。
- **本地化策略**: `scripts/core` 下的所有数据生成脚本现在直接读取用户本地安装的 PlatformIO 软件包 (`~/.platformio/packages/framework-arduinoststm32`)。这确保了生成的板卡数据与实际编译环境完全一致。
- **自动版本更新**: `npm run sync:stm32` 现在会自动执行 `pio pkg update -g -p ststm32`，确保在生成数据前获取官方最新的芯片定义。
- **配置修正**: 修复了 STM32WBA/U5 等新型号的 CPU 架构推断逻辑 (Cortex-M33)，并使运行时生成器优先使用数据驱动的构建配置。
- **核心修复**：解决了 STM32WBA 系列在 PlatformIO/STM32duino 环境中被误识别为 WB 系列导致的编译失败问题。
- **构建拦截器**：引入 `fix_wba_build.py` (v14) 自动修正编译宏定义与包含路径。
- **自动集成**：修改 `electron/config/templates.ts`，使软件在生成 WBA 项目配置时自动注入修复脚本。
- **变体修正**：修复了本地自定义 Variant 中的板级定义冲突，确保 symbols 链接正确。

#### 打包轻量化
- **移除冗余资源**: 安装包生成过程不再包含 600MB+ 的 Arduino Core 副本。
- **依赖托管**: EmbedBlocks Studio 现在直接调用系统环境中的 PlatformIO 资源进行编译，显著减小了安装包体积并消除了版本冲突风险。

## 技术细节

### API变更

**旧API (已弃用)**:
```typescript
arduinoGenerator.definitions_['key'] = code;
```

**新API (推荐使用)**:
```typescript
// 对于变量对象
arduinoGenerator.addVariable('key', code);

// 对于函数/结构体
arduinoGenerator.functions_['key'] = code;
```

### 影响范围

- **向后兼容**: ✅ 完全兼容
- **破坏性变更**: ❌ 无
- **运行时影响**: ❌ 无

迁移指南

对于新代码：
1. 使用 `addVariable` 代替 `definitions_` (对于对象)
2. 使用 `functions_` 代替 `definitions_` (对于函数/结构体)
3. 参考 `重构最佳实践手册.md` 了解最佳实践

对于现有代码：
- 无需立即迁移
- 可在方便时逐步更新
- variables.ts等特殊模块保持现状
