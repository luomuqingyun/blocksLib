# 更新日志 (CHANGELOG)

## [1.2.0] - 2026-02-01

### 🚀 新特性 (Features)

#### 输入焦点保护系统 (Input Focus Protection System)
- **全面输入检测**: 新增 `src/utils/input_protection.ts` 工具库，提供统一的输入焦点检测 API。
- **多场景覆盖**: 覆盖 7 种输入场景: INPUT/TEXTAREA/SELECT、contenteditable、ARIA role、Monaco Editor、对话框、自定义组件。
- **Blockly 快捷键修补**: 重构 `useBlocklyShortcuts.ts`，防止 Blockly 错误拦截非 Blockly 输入框的键盘事件。
- **智能验证延迟**: 优化 `useBlocklyValidation.ts`，字段变更时使用 800ms 长延迟，避免焦点丢失。

#### STM32 板卡配置优化 (STM32 Board Config Optimization)
- **自动数据扫描**: 重构 `src/config/stm32_boards.ts`，使用 Vite 的 `import.meta.glob` 自动扫描 `src/data/boards/stm32/` 目录。
- **单一数据源**: 删除冗余的 `stm32_series_data.json`，芯片数据统一从 JSON 文件生成。
- **完整数据保留**: 新增 `_rawData` 字段，保留芯片的完整信息 (封装、引脚图、外设配置) 供高级功能使用。
- **1430 芯片支持**: 自动识别并加载所有 STM32 系列的 1,430 个权威芯片数据。

### 🛠️ 重构 (Refactoring)

- **代码注释中文化**: 完成 `useBlocklyShortcuts.ts`、`useBlocklyValidation.ts`、`input_protection.ts` 的全量中文注释。
- **类型安全增强**: 修复 `Board.name` 类型为 `string | I18nString` 时的排序错误。

### 📝 文档 (Documentation)

- 更新 `更新日志.md` 记录本次架构优化。
- 更新 `docs/历史修改备忘录.md` 添加详细技术细节。

## [1.1.2] - 2026-01-15

### 🚀 新特性 (Features)
- **多串口支持**: 新增多串口支持 (Serial1/Serial2...)，自动适配板卡资源，支持特定硬件串口选择。
- **串口列表优化**: 串口列表增加 "Default" 标识，并按数字顺序 (Serial1, 2, 3...) 智能排序。
- **I2C 通信模块**: 新增 I2C 通信模块 (初始化、读写、设备扫描)。
- **SPI 通信模块**: 新增 SPI 通信模块 (初始化、高速传输)。

- **芯片信息模块 (Chip Info)**: 新增获取 MAC 地址、CPU 主频及 Flash 容量的跨平台积木。

### 🐞 修复 (Fixes)
- **生成器稳定性**: 修复了 `addVariable` 方法缺失导致的构建崩溃及输入框焦点丢失问题。
- **未连接板卡**: 修复了未连接板卡时生成器可能崩溃的问题。
- **类型安全**: 完善了 `ArduinoGeneratorExtensions` 接口定义。

## 技术细节

### API变更

**旧API (已弃用)**:
```typescript
arduinoGenerator.definitions_['key'] = code;
```

**新API (推荐使用)**:
```typescript
// 对于变量对象
arduinoGenerator.addVariable('key', code);

// 对于函数/结构体
arduinoGenerator.functions_['key'] = code;
```

### 影响范围

- **向后兼容**: ✅ 完全兼容
- **破坏性变更**: ❌ 无
- **运行时影响**: ❌ 无

迁移指南

对于新代码：
1. 使用 `addVariable` 代替 `definitions_` (对于对象)
2. 使用 `functions_` 代替 `definitions_` (对于函数/结构体)
3. 参考 `重构最佳实践手册.md` 了解最佳实践

对于现有代码：
- 无需立即迁移
- 可在方便时逐步更新
- variables.ts等特殊模块保持现状
