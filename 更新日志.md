## [1.4.2] - 2026-03-10

### 🐞 全局输入冻结彻底根治 (Global Input Freeze Eradication)
- **Body Hide 致命重绘移除**: 删除了 `App.tsx` 中在每次窗口获焦时将 `document.body.style.display` 设为 `none` 的危险操作。此操作会**立即销毁所有输入框的焦点状态**，是导致全局输入失灵的首要元凶。
- **IPC 写入频率降频 100 倍**: 将 `InputLoggerService` 的持久化阈值从每 5 次交互提升至每 500 次，并移除了每次点击和焦点变化时的即时文件写入，大幅缓解了 IPC 通道拥堵。
- **Blockly DOM 操控简化**: `RightPanel.tsx` 中的标签页切换逻辑不再直接操控 `.blocklyWidgetDiv` 的 DOM 属性（`display`, `pointerEvents`），改为仅调用 Blockly 官方 API `hideChaff()`，避免了与 Blockly 内部状态管理的冲突。
- **同步原生弹窗阻塞根除**: 修复了在欢迎页「最近项目列表」点击恢复异常项目时，由于遗漏的老旧原生 `confirm()` 对话框同步阻塞主线程，导致 Chromium Compositor 焦点状态崩溃从而引发输入假死的顽疾。已全局替换为异步安全的 IPC 对话框接口。

### 🐞 Chromium Compositor 焦点脱节终极修复 (Chromium Compositor Focus Desync Fix)
- **六轮深度排查**: DOM 事件拦截 → z-index 遮挡 → capture 阶段拦截 → `mainWindow.on('focus')` → IPC 破坏性 blur→focus → **非破坏性三层防御 + 激进 DOM 焦点重置**。
- **Electron 三层防御** (`electron/main.ts`): (1) `mainWindow.on('focus')` → `webContents.focus()`；(2) IPC `app:focus-fix` → 非破坏性 `webContents.focus()`；(3) `did-finish-load` → 启动时自动同步 Compositor。
- **激进 DOM 焦点重置** (`AiAssistantPanel.tsx`): `onMouseDown` 执行 `el.blur()` → `requestAnimationFrame` → `el.focus()` → `el.setSelectionRange(len, len)`，强制 Chromium 渲染光标。
- **全键盘拦截**: `onKeyDown` 对所有按键执行 `stopPropagation()`，防止 Blockly 消费字符键。
- **未来复发指南**: 按 `Ctrl+Shift+L` 导出诊断日志 → 检查 BLUR/FOCUS 模式 → Alt+Tab 可修复则扩展 `focusFix()` 范围。

### ✨ AI 助手设置联动 (AI Tab Settings Integration)
- **设置即消失**: 当用户在设置中关闭「启用 AI 助手」开关后，右侧面板底部的 AI 标签页和对话面板将**立即隐藏**，不再占据界面空间。

### API变更

**旧API (已弃用)**:
```typescript
arduinoGenerator.definitions_['key'] = code;
```

**新API (推荐使用)**:
```typescript
// 对于变量对象
arduinoGenerator.addVariable('key', code);

// 对于函数/结构体
arduinoGenerator.functions_['key'] = code;
```

- **向后兼容**: ✅ 完全兼容

- **运行时影响**: ❌ 无

迁移指南

对于新代码：
1. 使用 `addVariable` 代替 `definitions_` (对于对象)

对于现有代码：
